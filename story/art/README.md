# ART — The Accrual World

> The illustration project for *The Accrual World*. This folder holds the **art
> direction** and the **image prompts** that bring the book to life inside the app.
> Every image is generated by a modern text-to-image model from a self-contained
> prompt = **a subject prompt + the shared STYLE SUFFIX**.

The goal: one cohesive, warm, storybook-meets-editorial look across the whole
book — character portraits, chapter covers, and spot illustrations — all tuned to
the app's palette (cream `#fbf6ee`, terracotta `#b5635a`, gold `#a88a5c`, plum-ink
`#2a1f2d`). The single source of truth for that look is **`STYLE.md`**. Read it
first.

> Only `STYLE.md` and this `README.md` exist right now. The `prompts/` subfolders
> are the next step — see *Conventions* below for how they should be named and
> structured so they stay consistent with the book.

## How to build a full prompt (the core mechanic)

Every final prompt is **two parts concatenated**:

```
[ SUBJECT PROMPT ]  +  [ STYLE SUFFIX from STYLE.md §8, with the right aspect ratio ]
```

and you also paste the **NEGATIVE PROMPT** (STYLE.md §9) into the model's negative
field.

1. **Write the subject prompt.** Vivid, specific, self-contained: *who/what,*
   *doing what,* *where,* *the one idea or emotion.* Name the characters and pull
   their look-bible line from `STYLE.md §7` so they stay recognizable. Don't
   restate the medium/palette/lighting — the suffix does that.
2. **Append the STYLE SUFFIX** verbatim (`STYLE.md §8`), and set the bracketed
   **aspect ratio** to match the image type (3:4 / 16:9 / 1:1).
3. **Paste the NEGATIVE PROMPT** (`STYLE.md §9`) into the negative field.
4. Generate at the largest resolution offered for that ratio; check against the
   **consistency checklist** (`STYLE.md §10`) before accepting.

### Worked example

**Subject prompt (a Chapter 4 spot — the split):**

```text
The Allocator, a fate-like robed figure, gently divides the small glowing
terracotta being Sal into two: Lyle, a brighter, sharper, confident
terracotta-and-gold half holding a glowing download key, steps proudly to the
right; Sunny, a softer dimmer figure tinted cool deferred blue-mauve, is drawn to
the left toward a low arched dungeon doorway labelled with faint ledger marks. A
single gold seam of light runs between the two halves where Sal was one. Quiet,
bittersweet, sacred. Generous cream space around the three figures.
```

**Then append** the STYLE SUFFIX with `[aspect ratio: 1:1 square]`, **and** paste
the NEGATIVE PROMPT. That's a complete, ready-to-generate image.

## The three image types (and when to use each)

| Type | Ratio | Lives in | Purpose |
|---|---|---|---|
| **Portrait** | **3:4** | `prompts/portraits/` | A single cast member (or inseparable pair) on a simple tinted-paper ground. Maximal personality. |
| **Cover** | **16:9** | `prompts/covers/` | The wide establishing image of a chapter's *station*. Place + mood lead; leave a calm zone for a title. |
| **Spot** | **1:1** | `prompts/spots/` | The in-chapter teaching/emotion beat — one concept or moment, clean and isolated. The workhorses. |

Full composition logic for each is in `STYLE.md §6`.

## Naming & aspect conventions

Keep filenames lowercase, hyphenated, and prefixed so type and order are obvious
at a glance. Generated image files should mirror their prompt's name.

- **Portraits:** `portrait-<character>.{md,png}`
  e.g. `portrait-sal`, `portrait-friar`, `portrait-iris`, `portrait-lefty-righty`.
- **Covers:** `cover-<NN>-<station-slug>.{md,png}` (NN = `00` Prologue … `15`)
  e.g. `cover-00-prologue`, `cover-01-genesis`, `cover-04-revenue-labyrinth`,
  `cover-10-trinity-hall`, `cover-15-public-stage`.
- **Spots:** `spot-<NN>-<beat-slug>.{md,png}`
  e.g. `spot-04-the-split`, `spot-05-fifo-vs-lifo`, `spot-07-lefty-righty-bicker`,
  `spot-13-the-ferryman`.

**Aspect is implied by the prefix** (`portrait` → 3:4, `cover` → 16:9, `spot` →
1:1) and **must also be set explicitly** in the STYLE SUFFIX's bracket — the file
prefix is for humans; the bracket is what the model reads.

### Suggested per-prompt entry shape (in the `prompts/*` files)

```md
## spot-04-the-split
- **type:** spot (1:1)
- **chapter:** 4 — The Labyrinth of Revenue
- **beat:** the Allocator splits Sal into Lyle and Sunny
- **characters:** Sal, Lyle, Sunny, the Allocator   (look-bible: STYLE.md §7)
- **prompt:** <the self-contained subject prompt>
```

The final string fed to the model is always **that `prompt` + the STYLE SUFFIX +
the right aspect bracket**, with the NEGATIVE PROMPT in the negative field.

## Rules of the road (so the book stays one book)

1. **`STYLE.md` is law for the look.** Never re-describe medium/palette/light in a
   subject prompt — append the suffix instead. One illustrator's hand throughout.
2. **Pull character looks from `STYLE.md §7`** verbatim-close, every time, so the
   cast is recognizable across chapters.
3. **Match the palette to the app** (cream/terracotta/gold/plum-ink); sage and
   brick are spices only.
4. **Honor the meaning-colors:** heroes & "value" lean terracotta/gold;
   liabilities & limbo lean cool mauve; danger/the GAAP chasm is deep brick; a
   balanced/true entry can carry a soft gold glow.
5. **One idea per image.** Cream negative space is a feature, not waste.
6. **No text in images** unless explicitly specified (models garble it); if a
   single word is unavoidable, specify it exactly and verify it.
7. **Respect the one-way road.** Cover art for a station should feel like the next
   stop on a chronological journey, not a random fantasy locale.
8. **Always run the `STYLE.md §10` checklist** before accepting an image.

## Where the subjects come from

The book's beats, stations, and cast are defined in the planning docs one level up:

- `../01-STORY-BIBLE.md` — world, full cast, themes, running gags (the source for
  *who* and *what feeling*).
- `../00-OUTLINE.md` — the 16 chapters and their stations + story beats (the source
  for *which images each chapter needs*).
- `../NUMBERS.md` — the figures, if a number must appear in an image.

When writing a new subject prompt, start from the chapter's brief in `00-OUTLINE.md`
(its station = the cover; its key beats = the spots), then dress it in `STYLE.md`.
